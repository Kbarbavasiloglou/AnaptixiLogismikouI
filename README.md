# ΣΥΣΤΗΜΑ ΕΘΕΛΟΝΤΙΣΜΟΥ ΚΑΙ ΔΩΡΕΩΝ

## 1. Περιγραφή

Η παρούσα εφαρμογή υλοποιεί το use case "Πραγματοποίηση Δωρεάς

Ο απλός χρήστης του συστήματος (Donor) μπορεί να προχωρήσει στις παρακάτω ενέργειες:

-εισαγωγή των στοιχείων του

-επιλογή ποσού δωρεάς

-δήλωση τρόπου πληρωμής

-καταχώρηση ημερομηνίας δωρεάς

κι ανάλογα λαμβάνει μήνυμα επιτυχίας και απόδειξης πληρωμής ή αποτυχίας

Ο διαχειριστής (Admin user) μπορεί να συνδεθεί στο σύστημα και να δει αναφορά με όλες
τις καταγεγραμμένες δωρεές.

## 5. Δομή του κώδικα

Σύντομη περιγραφή των αρχείων:

```text
Anaptixi-Logismikou-1/      # Ρίζα του project
│
├── main.py              # Είσοδος του προγράμματος / μενού / διάλογος με χρήστη
├── models.py            # Κλάσεις οντοτήτων (Donor, Donation, PaymentReceipt)
├── services.py          # Λογική συστήματος / υλοποίηση use cases
└── diagrams/
    ├── use_case_diagram.png
    ├── class_diagram.png
    ├──sequence diagram.png
│
│
└── περιγραφή σεναρίου/
    └── περιγραφή σεναρίου.pdf
```

Τι κάνει το κάθε αρχείο:

 - `main.py`:
    - περιέχει το μενού της εφαρμογής,
    - διαβάζει input από τον χρήστη,
    - καλεί τις κατάλληλες συναρτήσεις από services.py.
    - Υλοποιεί τον έλεγχο σύνδεσης του διαχειριστή.

- `models.py`:
    - περιέχει τις βασικές κλάσεις του συστήματος (οντότητες).

- `services.py`:
    - Περιέχει τη λογική της εφαρμογής.
    - περιέχει συναρτήσεις που υλοποιούν τα βήματα του use case (δημιουργία δωρεάς, παραγωγή αναφοράς).
    - Διαχειρίζεται τη λίστα των δωρεών.

## 6. UML Διαγράμματα

Στον φάκελο diagrams/ περιλαμβάνονται:

 - `use case diagram` - use case diagram του συστήματος, μέρος του οποίου είναι η παρούσα υλοποίηση

 - `class_diagram.png` – class diagram των βασικών κλάσεων που υλοποιούνται.

 - `sequence_diagram.png` – sequence diagram για το βασικό use case, που περιγράφει τη συνολικότερη υλοποίηση (μαζί με την επιλογή των τρόπων πληρωμής και την αποστολή sms στον δωρητή.

## 8. Γνωστά Περιοριστικά Σημεία / Μελλοντική Επέκταση

-Τα δεδομένα αποθηκεύονται προσωρινά στη μνήμη. Μέσα στη main υπάρχουν δεδομένα δωρεών αποθηκευμένα για να μην είναι κενή η αναφορά.

-Δεν χρησιμοποιείται Βάση Δεδομένων.

-Σε επόμενη έκδοση μπορούν να υλοποιηθούν οι έλεγχοι για την πληρωμή (στοιχεία κάρτας, έγκριση πληρωμής από την τράπεζα του καρτούχου κοκ), καθώς και η αποστολή sms για ενημέρωση σχετικά με την κατάθεση της δωρεάς.

-Επίσης μπορούν να προστεθούν έλεγχοι για την μορφή των δεδομένων που εισάγονται (π.χ. ημερομηνία, όνομα, mail).

Επίσης μπορεί να προστεθεί φίλτρο στην αναφορά, σχετικά με το εύρος των ημερομηνιών, καθώς και τα δεδομένα να αποθηκεύονται σε JSON αρχεία, ώστε να είναι λειτουργικό το πρόγραμμα όσο αυξάνονται οι καταχωρήσεις.
